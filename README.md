# VOFF — Cyber Troopers Virtual On

*A 154 MB CD-ROM from 1997, reverse-engineered into a native Linux binary through 14 diary entries, 4,162 decompiled functions, and a 5-stage build pipeline.*

---

## What happens when you type `make`

```
  von.img ──► 626 files extracted ──► 4,162 functions decompiled ──► voff_sdl
  (154 MB)     (ISO 9660, ~30s)       (Ghidra headless, ~30min)       (native ELF, ~10s)
```

You place the original `von.img` CD-ROM image in the directory and run `make sdl`.
Everything else is automatic. The pipeline:

1. **Stage 1** — extracts 626 files from the ISO 9660 filesystem (game assets,
   raw PCM audio, V_ON.EXE)
2. **Stage 2** — imports the 6.4 MB PE32 executable into Ghidra, runs full
   auto-analysis (~3 minutes, all functions recovered)
3. **Stage 3** — decompiles every function to C, dumps the 50 MB data section
   and 301 KB read-only data section
4. **Stage 4** — builds a native 64-bit Linux binary (`voff_sdl`) that loads
   the original game data at runtime

No copyrighted material is distributed. Everything is regenerated from the
original CD-ROM.

## Two ways to run

| Binary | Runtime | Size | What it does |
|---|---|---|---|
| `voff_sdl` | **Native Linux** — SDL2 + OpenGL | 4.1 MB | Full game engine, no Wine, no Win32 |
| `voff.exe` | Wine (Winelib) | 422 KB | Uses Wine's D3D3 → OpenGL translation |

```bash
./run_voff.sh --sdl        # native, no Wine
./run_voff.sh              # Winelib, needs distrobox
```

animation, render, sync).

## Quick start

```bash
git clone https://github.com/longjoel/voff.git
cd voff

# 1. Place the original CD-ROM image
cp /path/to/von.img .

# 2. Install Ghidra 11.3.1
#    Download: https://github.com/NationalSecurityAgency/ghidra/releases
#    Extract to: ghidra/ghidra_11.3.1_PUBLIC/

# 3. Run the pipeline
make stage1    # ISO extraction (~30s)
make stage2    # Ghidra analysis (~3min)
make stage3    # Decompile all functions (~30min)
make sdl       # Build native binary (~10s)
make run-sdl   # Run
```

**Winelib build** (Wine + D3D3, needs distrobox):
```bash
distrobox create -n voff-builder -i fedora:44
distrobox enter voff-builder -- sudo dnf install -y wine-devel gcc gcc-c++ SDL2-devel
make winelib
./run_voff.sh
```

## Why?

Because in 2026, a 1997 PC port of a Sega arcade game shouldn't need a Windows
98 VM with S3 ViRGE drivers to run. It should compile from source and launch
with zero friction.

Built with [Ghidra](https://ghidra-sre.org/), SDL2, OpenGL, Wine, and far too
much caffeine.
